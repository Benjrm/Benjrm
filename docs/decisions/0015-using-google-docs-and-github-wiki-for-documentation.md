# Using Google Docs and GitHub Wiki for Documentation

## Context and Problem Statement

The team needs a way to create and maintain documentation throughout the project, covering both collaborative working documents and the final project documentation.

## Decision Outcome

Google Docs is used for all collaborative, work-in-progress documents such as sprint notes, Definition of Ready (DoR), Definition of Done (DoD), GDPR-related documents, and project presentations. It allows multiple team members to write and edit simultaneously and is already part of everyone's daily workflow.

GitHub Wiki is used for the final project documentation. Hosting the documentation directly in the repository keeps it close to the codebase and ensures it remains accessible alongside the source code without requiring a separate platform.
